# Cas d'usage

## Qu'est-ce que c'est ?

- Schématisation de la manière dont un utilisateur interagit avec le système pour accomplir une tâche précise

## A quoi ça sert ?

- Illustrer toutes les fonctionnalités
- Vérifier que tous les besoins exprimés soient bien satisfaits
- Sert de base pour la validation des tests fonctionnels
- Permet d'identifier quel sont les acteurs principaux (a gauche) et secondaires (a droite) (dans les schémas UML)

## Ce qu'on peut retrouver dedans

- Les acteurs (utilisateurs & rôles)
- Les cas d'usage
- Les dépendances entre cas d'usage
