# Plan de test d'intégration

## Qu'est-ce que c'est ?

- Document qui définit la stratégie de test des composants assemblés.
- Il couvre les enchaînements métier et les interfaces entre modules.

## A quoi ça sert ?

- Vérifier la cohérence du fonctionnement global du système.
- Valider les échanges entre briques techniques.
- Réduire les risques de régression avant validation fonctionnelle.

## Ce qu'on peut retrouver dedans

- Les scénarios métier d'intégration à exécuter.
- Les dépendances techniques entre composants.
- Les jeux de données et environnements de test.
- Les critères de succès et de sortie de phase.
