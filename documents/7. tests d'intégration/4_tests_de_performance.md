# Tests de performance

## Qu'est-ce que c'est ?

- Document qui formalise les tests de charge, de temps de réponse et de stabilité.
- Il mesure le comportement du système sous différentes contraintes d'usage.

## A quoi ça sert ?

- Vérifier que les performances respectent les attentes du projet.
- Détecter les goulets d'étranglement techniques.
- Préparer la mise en production avec des seuils mesurés et validés.

## Ce qu'on peut retrouver dedans

- Les scénarios de performance (charge nominale, pic, endurance).
- Les indicateurs suivis (latence, débit, erreurs, ressources).
- Les seuils cibles et résultats mesurés.
- Les actions d'optimisation recommandées.
