# Scripts / Automation pour les tests

## Qu'est-ce que c'est ?

- Document qui décrit les scripts et outils d'automatisation des tests unitaires.
- Il précise comment lancer les tests automatiquement et interpréter les résultats.

## A quoi ça sert ?

- Exécuter les tests rapidement et de manière reproductible.
- Réduire les erreurs humaines liées aux tests manuels.
- Faciliter l'intégration des tests dans une chaîne CI/CD.

## Ce qu'on peut retrouver dedans

- Les commandes de lancement des tests unitaires.
- Les scripts utilisés (bash, npm, phpunit, etc...).
- Les prérequis d'exécution (versions, dépendances, variables).
- Les rapports générés (couverture, logs, résultats détaillés).
